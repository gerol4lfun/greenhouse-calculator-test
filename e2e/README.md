# E2E: раздел «Редактирование заказа»

Автопрогон этапов 1–2 (поиск, список, открытие редактирования) по плану из проекта «тг бот заказы теплицы».

## Что нужно

1. **Сервер** — приложение должно быть доступно по `http://localhost:3000` (например: `npx serve -p 3000 .` в корне greenhouse-calculator-main).
2. **Авторизация** — логин и пароль от приложения (экран «Вход в систему»). **Пароль не коммитить и не писать в репозиторий.**
3. **Тестовый телефон** — номер, по которому в Supabase есть хотя бы один заказ (для проверки 1.3 и 2.1–2.4).

## Откуда брать данные

- **Телефоны с заказами:** выгрузка из Supabase `orders_rows-5.csv` (лежит в Загрузках или в проекте «тг бот»). Формат: CSV, заголовок в первой строке, столбец **client_phone** (11 цифр, с 7). Для E2E подойдёт любой телефон из этого столбца, например `71111111111` или `77777777777`.
- **Логин и пароль:** только из локального `.env` или переменных окружения при запуске. В репозитории хранить только шаблон (см. ниже).

## Запуск

```bash
# В корне greenhouse-calculator-main

# 1) Поднять сервер (в отдельном терминале или в фоне)
npx serve -p 3000 .

# 2) Запустить тест с переменными из .env или явно:
# Через .env (рекомендуется — пароль не в истории)
# Скопируй e2e/.env.example в e2e/.env и заполни TEST_LOGIN, TEST_PASSWORD.
# Затем:
source e2e/.env 2>/dev/null || true
TEST_PHONE=71111111111 npm run test:e2e

# Или одной строкой (пароль в истории термина — только для локального запуска):
TEST_PHONE=71111111111 TEST_LOGIN=admin TEST_PASSWORD='твой_пароль' npm run test:e2e
```

Подставь вместо `71111111111` любой **client_phone** из `orders_rows-5.csv`.

## Результаты

- В консоли: прошёл/не прошёл тест.
- Файл `e2e/e2e-results.md` — таблицы по этапам 1–9 (этапы 1–2 заполняются по факту прогона, 3–9 — «Ручная проверка»).
- Для отчёта в «тг бот заказы теплицы» результаты переносятся в **РЕЗУЛЬТАТЫ_ТЕСТОВ_РЕДАКТИРОВАНИЯ.md** по форме этого файла.

## Переменные окружения

| Переменная       | Описание |
|------------------|----------|
| `BASE_URL`       | URL приложения (по умолчанию `http://localhost:3000`). |
| `TEST_PHONE`      | Телефон с заказами в Supabase (11 цифр, с 7). Без него — случайный номер, шаги 1.3 и 2.x не выполняются. |
| `TEST_LOGIN`      | Логин для входа в приложение. |
| `TEST_PASSWORD`   | Пароль. **Не коммитить.** Задавать только в `.env` или при ручном запуске. |
