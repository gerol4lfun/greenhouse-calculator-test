# 📚 База знаний (Knowledge Base) - Документация

## 🎯 Что это?

База знаний для менеджеров — это структурированная база данных с фактами и инструкциями по работе с клиентами. Помогает менеджерам быстро находить ответы на вопросы клиентов и отрабатывать возражения.

## 📋 Структура данных

Каждая карточка в базе содержит:

- **id** — уникальный идентификатор (например, "GH-0001")
- **type** — тип: `FACT` (факт) или `HOWTO` (инструкция)
- **audience** — аудитория: `internal_only` (только для менеджеров) или `client_safe` (можно показывать клиенту)
- **title** — краткий заголовок
- **text** — основной текст карточки
- **tags** — массив тегов для поиска
- **source_ref** — ссылка на источник (JSON с post_id, date, line_no)
- **deprecated** — помечена ли как устаревшая
- **kb_version** — версия базы знаний
- **kb_updated_at** — дата обновления

## 🚀 Установка

### ШАГ 1: Создать таблицу

1. Откройте Supabase Dashboard
2. Перейдите в **SQL Editor**
3. Откройте файл `db/migrations/20260203_create_knowledge_base.sql` (рекомендуется) или `sql/supabase_setup_knowledge_base.sql`
4. Скопируйте весь SQL код
5. Вставьте в SQL Editor
6. Нажмите **Run** (или F5)

**✅ Готово! Таблица создана.**

### ШАГ 2: Загрузить данные

**Вариант 1: Через Supabase Dashboard (РЕКОМЕНДУЕТСЯ)**

1. Откройте Supabase Dashboard
2. Перейдите в **Table Editor**
3. Выберите таблицу `knowledge_base`
4. Нажмите **Import data** (или **Insert** → **Import from file**)
5. Выберите файл `knowledge_base_v1.json`
6. Подтвердите импорт

**Вариант 2: Через SQL (если JSON не поддерживается)**

1. Откройте файл `knowledge_base_v1.json`
2. Для каждой записи создайте INSERT запрос (см. пример в `load_knowledge_base_data.sql`)
3. Выполните все запросы в SQL Editor

**✅ Готово! Данные загружены.**

## 📖 Использование

### Для менеджеров:

- **client_safe** — можно копировать текст и отправлять клиенту
- **internal_only** — только для внутреннего использования, не показывать клиенту

### Поиск по тегам:

- Используйте теги для быстрого поиска нужной информации
- Например: `возражение_дорого`, `оплата`, `доставка`, `усиление`

### Фильтрация:

- По типу: `FACT` или `HOWTO`
- По аудитории: `internal_only` или `client_safe`
- По тегам: любой тег из массива

## 🔄 Обновление данных

### Правило "один файл — одна истина":

- Все правки обсуждаются где угодно
- Но в "продукт" попадает только то, что в файле `knowledge_base_v1.json`
- После обновления JSON файла — загрузите данные заново в Supabase

### Версионирование:

- Каждая карточка имеет поле `kb_version` (текущая версия: `v1.0`)
- При обновлении базы знаний меняйте версию
- Устаревшие карточки помечайте `deprecated: true`

## 🔍 Примеры запросов

### Получить все карточки для клиентов:
```sql
SELECT * FROM knowledge_base 
WHERE audience = 'client_safe' AND deprecated = false
ORDER BY title;
```

### Найти карточки по тегу:
```sql
SELECT * FROM knowledge_base 
WHERE 'возражение_дорого' = ANY(tags) AND deprecated = false;
```

### Получить все инструкции (HOWTO):
```sql
SELECT * FROM knowledge_base 
WHERE type = 'HOWTO' AND deprecated = false
ORDER BY title;
```
