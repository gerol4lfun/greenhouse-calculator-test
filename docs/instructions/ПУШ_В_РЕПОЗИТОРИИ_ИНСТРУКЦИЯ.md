# Инструкция: как правильно отправлять изменения в репозитории

**Для кого:** Cursor (агент), разработчики.  
**Важно:** у проекта два разных репозитория. Код из одной папки пушится в два разных remote — без костылей и без путаницы.

---

## Два репозитория (не один)

| Репозиторий | Назначение | Где лежит код в этом проекте | Куда пушить |
|-------------|------------|------------------------------|-------------|
| **gerol4lfun/greenhouse-calculator** | Калькулятор теплиц (весь сайт, JS, FAQ, документация). Деплой: GitHub Pages и т.п. | Вся папка проекта (корень, `js/`, `css/`, `docs/`, `telegram-bot/` и т.д.) | `origin` в этой же папке (если она — клон этого репо) |
| **gerol4lfun/delivery-bot-telegram** | Telegram-бот, который обновляет даты доставки в Supabase. Деплой: Railway из этого репо. | Только папка **`telegram-bot/`** внутри проекта калькулятора | Отдельный клон репо `delivery-bot-telegram`; файлы копировать из `telegram-bot/` и пушить там |

Итого:  
- Калькулятор = один репо = пушим из папки проекта.  
- Бот = другой репо = исходники в `telegram-bot/`, в GitHub попадают только после копирования в клон `delivery-bot-telegram` и push из него.

## 1. Репозиторий калькулятора (greenhouse-calculator)

**URL:** https://github.com/gerol4lfun/greenhouse-calculator

### Как должно быть (без костылей)

- Рабочая папка — **клон** этого репо (например `git clone https://github.com/gerol4lfun/greenhouse-calculator.git`).
- Внутри есть полный `.git` (в т.ч. `config` с `remote origin` и ветка `main`).
- Все правки делаются в этой папке. Отправка в GitHub — только из неё.

### Команды для отправки

```bash
# Перейти в корень проекта калькулятора (та папка, где лежат index.html, js/, telegram-bot/, docs/)
cd "/путь/к/greenhouse-calculator"

git status
git add .
git commit -m "vXXX: краткое описание изменений"
git push origin main
```

### Если папка не клон (например, распакован ZIP или скопирована без .git)

- Либо заново клонировать репо и перенести в него изменённые файлы, затем `add` / `commit` / `push` из клона.
- Либо в этой папке восстановить `.git`: как минимум нужны `.git/config` с `[remote "origin"] url = https://github.com/gerol4lfun/greenhouse-calculator.git` и рабочие refs (например после `git init` и `git fetch origin` сделать `git reset --soft origin/main`, потом снова `add` / `commit` / `push`).

Большой push (много файлов): при ошибке 400 можно увеличить буфер:
`git config http.postBuffer 524288000`

## 2. Репозиторий Telegram-бота (delivery-bot-telegram)

**URL:** https://github.com/gerol4lfun/delivery-bot-telegram  

**Назначение:** бот для обновления дат доставки (получает сообщения в ТГ → парсит → обновляет Supabase). Railway деплоит именно из этого репо.

### Важно

- Код бота в проекте калькулятора лежит в папке **`telegram-bot/`**.
- В репо **delivery-bot-telegram** на GitHub пушатся **только** файлы, скопированные из этой папки. Отдельная папка = отдельный репо, общих коммитов нет.

### Какие файлы отвечают за бота и должны попасть в delivery-bot-telegram

Копировать из **`telegram-bot/`** в клон репо **delivery-bot-telegram**:

| Файл в проекте | Назначение |
|----------------|------------|
| `index.js` | Точка входа бота |
| `parser.js` | Парсер сообщений с датами |
| `cities.js` | Канонические названия городов |
| `supabase.js` | Работа с Supabase (даты доставки) |
| `package.json` | Зависимости |
| `Procfile` | Инструкция для Railway |
| `railway.json` | Конфиг Railway (если нужен) |
| `README.md` | Описание (по желанию) |

Минимально необходимый набор для работы бота: **index.js, parser.js, cities.js, supabase.js, package.json**. Остальное — по необходимости.

### Правильная последовательность (без костылей)

1. Клонировать репо бота в отдельную папку (один раз):
   ```bash
   git clone https://github.com/gerol4lfun/delivery-bot-telegram.git
   cd delivery-bot-telegram
   ```

2. Когда в проекте калькулятора меняли файлы в `telegram-bot/`, скопировать их в этот клон:
   ```bash
   # Путь к папке проекта калькулятора (где лежит telegram-bot/)
   CALC="/путь/к/greenhouse-calculator"
   BOT_REPO="/путь/к/delivery-bot-telegram"

   cp "$CALC/telegram-bot/index.js"    "$BOT_REPO/"
   cp "$CALC/telegram-bot/parser.js"   "$BOT_REPO/"
   cp "$CALC/telegram-bot/cities.js"   "$BOT_REPO/"
   cp "$CALC/telegram-bot/supabase.js" "$BOT_REPO/"
   cp "$CALC/telegram-bot/package.json" "$BOT_REPO/"
   # при необходимости:
   cp "$CALC/telegram-bot/Procfile"    "$BOT_REPO/"
   cp "$CALC/telegram-bot/railway.json" "$BOT_REPO/"
   ```

3. Закоммитить и отправить **из папки delivery-bot-telegram**:
   ```bash
   cd "$BOT_REPO"
   git add index.js parser.js cities.js supabase.js package.json
   git status
   git commit -m "Краткое описание: что изменено в боте"
   git push origin main
   ```

После пуша Railway (если подключён к этому репо) сам пересоберёт и задеплоит бота.

## Краткая памятка для Cursor

1. **Правки калькулятора** (index.html, js/scripts.js, docs, css, и т.д.) → пушить из **папки проекта** в репо **greenhouse-calculator** (`git add .` → `commit` → `push origin main`).
2. **Правки бота** (файлы в `telegram-bot/`) → сначала скопировать нужные файлы в **отдельный клон** репо **delivery-bot-telegram**, затем из этой папки сделать `git add` → `commit` → `push origin main`. В репо калькулятора бот не деплоится.
3. **Синхронизация версии (vXXX):** обновить версию в обоих репо — в greenhouse-calculator (package.json, README, js/scripts.js, docs, .version, telegram-bot/) и в delivery-bot-telegram (скопировать из `telegram-bot/` в клон, commit + push).
4. Два репо — два разных remote и две разные папки для push. Не путать.
