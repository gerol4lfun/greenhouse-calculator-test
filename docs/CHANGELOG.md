## Версия v269 — gift slots в edit: превью и восстановление

**Дата:** 2026-03-12.

- **Баг:** после «Рассчитать» в панели «Параметры теплицы» gift slots не обновлялись до сохранения позиции.
- **Фикс:** `getEditOrderCompositionTotalWithPreview()` — total с учётом lastModalCalculationResult; после «Рассчитать» вызывается `updateEditOrderGiftsBlock(previewTotal, true)`; при закрытии панели (отмена) — `lastModalCalculationResult = null` и `updateEditOrderGiftFromTotal()` для восстановления. Режим `previewOnly` не пишет в giftEl, чтобы при отмене не терять данные.
- **UX:** уведомление о смене количества подарков в edit/add panel переведено на заметный toast; tier-change теперь явно виден менеджеру.
- **UX:** кнопка «Рассчитать» переименована в «Обновить расчёт»; вторичный стиль; подсказка под кнопкой удалена.

**Зафиксировано на сегодня (12.03.2026):**
- commercial_offer при edit пересобирается корректно; для пересчитанных позиций источник — результат расчёта.
- cancelled заказ нельзя редактировать (UI + проверка перед save).
- платные допы не исчезают из long КП; edit платных допов: edit → save → Supabase → карточка → commercial_offer.
- gifts = slots; пороги 35k/55k/75k; макс 3 слота; автомат только от 55k при наличии форточки; автоматическая форточка = 2 слота.
- 3 форточки / 3 полива — можно выбрать; gift сохраняется и доезжает в Supabase; slot count от total; tier-change — toast.
- gift thresholds и gift names централизованы.

**Открытые хвосты:** ручной пересчёт vs live-preview; UX кнопки; UI gift в preview-карточке; архитектура giftCore — до прода не трогать.

**Документация (12.03.2026):** созданы docs/TRUTH_MAP.md, LEGACY_MAP.md, PREPROD_PLAN.md, SMOKE_CHECKLIST.md — карта проекта и план до прода.

## Версия v267 — commercial_offer при edit; запрет редактирования cancelled

**Дата:** 2026-03-12.

- **Business truth подарков:** зафиксирована в `docs/GIFT_TRUTH.md`: slot model, автомат от 55k, автоматическая форточка = 2 слота.

     - Для теплиц 6м+: центральная грядка 4м + 2 боковые по длине теплицы
     - Ширина зависит от ширины теплицы (>=3.5м: 0.8м центр/1м бока, иначе: 0.5м центр/0.8м бока)
   - Два варианта добавления: "Без сборки" и "Со сборкой"
   - Автоматическое закрытие модального окна после выбора рекомендуемого варианта
   - Автоматический пересчет стоимости при добавлении грядок

2. **Цены на грядки**
   - Данные хранятся локально в массиве `BEDS_DATA` (40 вариантов: низкие/высокие × 4 ширины × 5 длин)
   - Цены одинаковые для всех городов (захардкожены в коде)
   - Ширины: 0.5м, 0.65м, 0.8м, 1м
   - Длины: 4м, 6м, 8м, 10м, 12м
   - Высоты: 19см (низкие), 38см (высокие)

3. **Сборка грядок**
   - Опциональная услуга с ценами в зависимости от длины теплицы:
     - 4м: 990 руб
     - 6м: 1490 руб
     - 8м: 1990 руб
     - 10м: 2490 руб
     - 12м: 2990 руб
     - 14м: 3490 руб
     - 16м: 3990 руб
   - Чекбокс "Добавить сборку грядок" в модальном окне
   - Динамический расчет стоимости сборки (за 1 грядку или за все грядки)

4. **Интеграция в коммерческое предложение**
   - Грядки добавляются в длинное КП: "Грядки оцинкованные: [тип] [размеры] - [цена] руб."
   - Сборка грядок: "Сборка грядок (за 1 грядку) - XXXX руб." или "Сборка грядок (за все грядки) - XXXX руб."
   - В коротком КП: "Грядки: XXXX рублей (со сборкой)" или "Грядки: XXXX рублей"
   - Грядки учитываются в общей стоимости теплицы

5. **Система уведомлений (Toast)**
   - Замена стандартных `alert()` на стильные toast-уведомления
   - Типы: success, error, warning, info
   - Автоматическое исчезновение через 4-5 секунд
   - Неблокирующие уведомления
