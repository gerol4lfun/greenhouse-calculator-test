# Инструкции по проекту «Калькулятор теплиц»

Один документ: правила правок, где что лежит, что обновлять при релизе. При конфликтах с другими документами — ориентироваться на этот файл.

**Дата актуализации:** 15.03.2026  
**Текущая версия:** v271 (см. `js/scripts.js` → `APP_VERSION`; синхронизированы index.html ?v=, package.json _projectVersion, docs/CHANGELOG.md)

---

## 1. Правила правок

- **Правки только в `greenhouse-calculator-main`** — не в `archive/` внутри бота и не в других копиях.
- **Основные файлы:** `index.html`, `js/scripts.js`, `css/styles.css`. Всё остальное — данные и документация.
- **Перед правкой логики расчёта/КП** — открыть `docs/context/КОНТЕКСТ_ПРОЕКТА.md`: там карта всех функций и где что менять.
- **Supabase** — хранит заказы. Структура таблицы `orders` описана в `db/migrations/`. Изменения схемы — только через новый файл миграции.
- **Цены** — только в `js/scripts.js`. После изменения цен обязательно поднять `?v=` у css/js в `index.html`, чтобы менеджеры получили свежий кеш (сказать им «перезайдите»).
- **FAQ** — редактировать файлы в `docs/faq/`. Не переносить FAQ-контент в код.

## 3. Что обновлять при релизе новой версии

### 3.1 Синхронизация версии (vXXX) — обязательно

При каждом релизе обновить версию в **одном коммите** во всех местах:

| Файл | Где менять |
|------|-----------|
| `js/scripts.js` | `const APP_VERSION = "vXXX"` (первая строка файла) |
| `index.html` | `?v=XXX` у тега `<link rel="stylesheet">` и `<script src="js/scripts.js">` |
| `package.json` | поле `"version": "X.XX.0"` |
| `favicon.version` | строка `# Версия проекта: vXXX` |
| `docs/CHANGELOG.md` | добавить новый раздел `## Версия vXXX` с описанием изменений |
| `.version` в папках | `теплицы фото/`, `video/products/`, `kb/`, `telegram-bot/`, `Доп. инф-ция 2/`, `image/`, `db/migrations/`, `backup/`, `Инструкции по сборке теплиц/` |
| `README.md` | версия в заголовке или описании |

### 3.2 Если менялся бот дат доставки (`telegram-bot/`)

Бот живёт в отдельном репо. После правок скопировать файлы и запушить отдельно — полная инструкция: `docs/instructions/ПУШ_В_РЕПОЗИТОРИИ_ИНСТРУКЦИЯ.md`.

### 3.3 КОНТЕКСТ_ПРОЕКТА.md

При значимых изменениях архитектуры или появлении новых функций — обновить `docs/context/КОНТЕКСТ_ПРОЕКТА.md`: дату, версию, описание функции или файла. Это главный ориентир при потере контекста.

## 5. Обновление цен и данных

| Что | Где в коде | Что сделать после |
|-----|-----------|-------------------|
| Цены теплиц/допов | `js/scripts.js` (константы в начале файла) | Поднять `?v=` в `index.html`; сказать менеджерам «перезайдите» |
| Даты доставки | Telegram-бот или Supabase напрямую | — |
| FAQ ответы | `docs/faq/FAQ_*.md` | При значимых правках поднять версию и запушить |
| Фото теплиц | `теплицы фото/` | Обновить `.version` в папке |
